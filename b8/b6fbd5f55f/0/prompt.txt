## Summary

Adds a new `attach` command accessible via `entire attach <session-id>`. Enables users to retroactively capture an agent session that wasn't tracked (ie if the agent was created outside of the project's root directory).

## Progress

* Has been tested and validated with Claude + Gemini. Testing support for other agents next.

### What it does

* Resolves the agent's transcript file from the session ID, reads it, and extracts modified files and the first user prompt
* Creates a checkpoint on the shadow branch with the session's transcript and file changes
* Condenses the checkpoint to `entire/checkpoints/v1` and transitions the session to IDLE
* Offers to amend the last commit with the `Entire-Checkpoint` trailer (interactive prompt, or `--force` to skip)
* Includes a `-a` or `--agent` flag to specify which agent. As with other commands, defaults to Claude as the agent.

### Key decisions

* **Transcript search across subdirectories**: Claude Code uses cwd (not repo root) for its project directory, so a session started from a subdirectory stores its transcript under a different project dir name. `attach` searches all project dirs that are subdirectories of the repo root as a fallback.
* **Agent-agnostic via `--agent` flag**: Defaults to `claude-code` but supports all registered agents (`gemini`, `opencode`, `cursor`, `copilot-cli`, `factoryai-droid`). The agent interface's `GetSessionDir` / `ResolveSessionFile` / `ReadTranscript` methods handle the per-agent differences.
* **No pre-state baseline**: Unlike hook-captured sessions that know the pre-session untracked files, `attach` has no pre-state. It uses `filterToUncommittedFiles` to limit checkpoint contents to files that are actually uncommitted, avoiding noise from already-committed changes.
* **Amend prompt with `--force`**: By default, shows the last commit and asks whether to amend it with the checkpoint trailer. `--force` skips the prompt and amends immediately. If the commit already has a trailer, it's a no-op.
* **Transcript helpers extracted to `transcript.go`**: `resolveTranscriptPath`, `searchTranscriptInSubdirs`, and related helpers are in a dedicated file to keep `attach.go` focused on the command logic.

## Test plan

* Unit tests for missing session ID, transcript not found, success path, duplicate session detection, and checkpoint ID output
* Manual test: `entire attach <real-session-id>` with Claude Code session
* Manual test: `entire attach --force <session-id>` amends commit without prompting
* Manual test: `entire attach --agent gemini <session-id>` with Gemini session
* Verify `entire rewind` works on attach-created checkpoints
